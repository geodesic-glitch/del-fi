WHAT IS THIS 
Drop documents into a folder. Connect a $30 radio. Your neighborhood now has an AI oracle -- no internet, no cloud, no accounts.

Del-Fi nodes sit on 900 MHz and 2.4 GHz meshes simultaneously, bridging coverage across your community. Every node added makes the whole net more useful. When someone sends a DM, Del-Fi finds the relevant docs, asks a local LLM, and replies within the 230-byte LoRa limit -- everything on your hardware.

Plug in sensors or cameras and the FactStore answers measurement queries directly -- no LLM, no hallucination on facts.

Drop documents into knowledge/, run --build-wiki once, and the LLM compiles them into a structured wiki. At query time it searches the wiki -- not raw docs -- so even a 1B model gets clean, focused context. The wiki compounds: each new source updates existing pages without losing what came before.

One package. Four dependencies. Fully offline.

HARDWARE 
COMPUTE (pick one)                         RADIO

  Raspberry Pi 5 8GB .... ~$80            Any Meshtastic or
  Jetson Orin Nano ...... ~$249           MeshCore LoRa radio.
  Mac Mini M4 ........... ~$499           Heltec V3 is ~$20.

  3B models are the sweet spot.              Antenna placement matters
  5-10s per response on a Pi.              more than radio choice.
QUICK START 
  $ curl -fsSL https://ollama.com/install.sh | sh
  $ ollama pull gemma4:4b && ollama pull nomic-embed-text
  $ git clone https://github.com/geodesic-glitch/del-fi.git && cd del-fi
  $ pip install -r requirements.txt
  $ cp config.example.yaml config.yaml  # set node_name and model
  $ python main.py --build-wiki  # compile knowledge/ → wiki/
  $ python main.py

  Edit config.yaml to set node_name, model, and personality.
  Drop .md or .txt files into knowledge/ and re-run --build-wiki.
  Everything else has sensible defaults.

  No radio? Test with:  python main.py --simulator

Trail Oracle
Solar node at a trailhead. Plant ID, trail conditions, emergency procedures.
Farm Oracle
Planting calendars, livestock medicine, equipment repair for the whole property.
Emergency Response
Triage protocols, shelter locations, phrase books. Works when towers don't.
The Dead Drop
Mysterious node appears on mesh. Cryptic name, oddly specific local knowledge.
Festival Concierge
Schedules, vendor maps, food guides. No cell service required.
Neighborhood Mesh
HOA rules, garbage day, business hours. "When's bulk trash pickup?"

ARCHITECTURE 
[LoRa Radio] <--serial/tcp--> [Mesh Adapter] <--> [Router] <--> [Tier Hierarchy]
                                     |               |               |
                              (meshtastic    [Formatter]    Tier 0  FactStore   sensor feed, no LLM
                               or meshcore)               Tier 1  WikiEngine  BM25 + LLM on compiled wiki
                                                           Tier 2  PeerCache   trusted peer Q&A
                                                           Tier 3  GossipDir   referrals only

main.py ................. entry point, daemon lifecycle, --build-wiki, --lint-wiki
del_fi/
  config.py ............. YAML loading, validation, oracle profiles
  core/
    knowledge.py ......... WikiEngine: compile knowledge/ → wiki/, BM25 query, lint
      knowledge/ .......... raw source docs (human-owned, gitignored)
      wiki/ ............... LLM-compiled pages, rebuilt via --build-wiki
    router.py ........... command dispatch, tier routing, !more buffer
    formatter.py ........ 230-byte enforcement, UTF-8 safety, chunking
    facts.py ............ FactStore: sensor feed, Tier 0 fast path
      cache/sensor_feed.json  external scripts write readings here
    peers.py ............ PeerCache (Tier 2) + GossipDirectory (Tier 3)
    memory.py ........... per-sender conversation history, TTL
    board.py ............ community message board, !post / !board
  mesh/
    base.py ............. abstract MeshAdapter interface
    meshtastic_adapter.py  Meshtastic radios (serial/TCP/BLE)
    meshcore_adapter.py    MeshCore radios (stub -- ready to implement)
    simulator.py ........ stdin/stdout for development
