WHAT IS THIS 
Drop documents into a folder. Connect a $20 radio. Your community now has an AI oracle that answers questions over LoRa mesh -- no internet, no cloud, no accounts.

Someone on the mesh sends your node a DM. Del-Fi searches your local docs, feeds the relevant chunks to a local LLM, and sends back a concise answer -- all within the 230-byte LoRa message limit. Everything runs on your hardware.

Connect sensors, trail cameras, or CV pipelines to the FactStore layer and Del-Fi answers measurement queries -- temperature, detections, current readings -- directly from structured data, bypassing the LLM entirely. No hallucination on facts.

Supports Meshtastic and MeshCore radios through a pluggable adapter system -- same oracle, your choice of radio stack.

Seven Python files. Four dependencies. That's it.

HARDWARE 
COMPUTE (pick one)                         RADIO

  Raspberry Pi 5 8GB .... ~$80            Any Meshtastic or
  Jetson Orin Nano ...... ~$249           MeshCore LoRa radio.
  Mac Mini M4 ........... ~$499           Heltec V3 is ~$20.

  3B models are the sweet spot.              Antenna placement matters
  5-10s per response on a Pi.              more than radio choice.
QUICK START 
  $ curl -fsSL https://ollama.com/install.sh | sh
  $ ollama pull qwen2.5:3b
  $ ollama pull nomic-embed-text
  $ git clone https://github.com/geodesic-glitch/del-fi.git
  $ cd del-fi && pip install -r requirements.txt
  $ cp config.example.yaml ~/del-fi/config.yaml
  $ python delfi.py

  That's 6 commands. Edit config.yaml to set node_name and model.
  Everything else has sensible defaults.

  No radio? Test with:  python delfi.py --simulator

Farm Oracle
Planting calendars, livestock medicine, equipment repair for the whole property.
Emergency Response
Triage protocols, shelter locations, phrase books. Works when towers don't.
The Dead Drop
Mysterious node appears on mesh. Cryptic name, oddly specific local knowledge.
Festival Concierge
Schedules, vendor maps, food guides. No cell service required.
Neighborhood Mesh
HOA rules, garbage day, business hours. "When's bulk trash pickup?"

ARCHITECTURE 
[LoRa Radio] <--serial/tcp--> [Mesh Adapter] <--> [Router] <--> [RAG Engine]
                                     |               |               |
                              (meshtastic    [Formatter]    [Mesh Knowledge]
                               or meshcore)   [FactStore]

delfi.py .............. entry point, daemon lifecycle, worker thread
config.py ............. YAML loading, validation, defaults
facts.py .............. FactStore: sensor / CV data, Tier 0 routing
  cache/sensor_feed.json  external scripts write readings here
  structured facts bypass the LLM entirely -- no hallucination on values
mesh/ ................. pluggable mesh protocol adapters
  base.py ............. abstract MeshAdapter interface
  meshtastic_adapter.py  Meshtastic radios (serial/TCP/BLE)
  meshcore_adapter.py    MeshCore radios (stub -- ready to implement)
  simulator.py ........ stdin/stdout for development
router.py ............. command dispatch, query pipeline, cache
rag.py ................ ChromaDB indexing, Ollama embeddings
formatter.py .......... markdown stripping, sentence truncation
meshknowledge.py ...... gossip, peer cache, referrals
